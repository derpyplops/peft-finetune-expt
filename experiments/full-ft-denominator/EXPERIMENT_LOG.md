- 2026-06-03: Premortem — wrote `reports/premortem.md`. Central insight: the dangerous failures here are silent-wrong-answer, not crashes. Top risks: (1.1) full FT at the paper's lr 5e-5 diverges/overfits → "% recovered" >100%; (1.2) generation eval doesn't string-match label vocab → denominator artificially ~0 on classification; (1.3) numerator (paper/W&B) vs our denominator computed in a drifted env / different metric choice → apples-to-oranges; (1.4) full FT < base on gsm8k/code → gain-normalized ratio undefined. Conclusion: the planned smoke test (cb/svamp/conala) is necessary but insufficient — it must additionally (a) inspect raw generations for label parsing, (b) include one large + one long-context (2048) run, (c) reproduce 1–2 of the paper's PEFT cells under our stack to validate the environment. Added go/no-go gates. This supersedes the smoke-test scope in plan.md Phase 2.
- 2026-06-03: Phase 0 (local prep, no GPU) — cloned PEFT-Factory + PEFT-Bench into `vendor/`, extracted the exact experimental setup from `scripts/peftbench/run_exp.sh`, the llama-3-8b-instruct example configs, `data/dataset_info.json`, and `compute_metrics.py`. **Key finding: NO subsampling** — configs load full HF `train` splits (no `max_samples` anywhere), so the earlier "~$150–600" estimate (which assumed subsampling) was wrong. Canonical run = 27 datasets, 5 seeds (42,123,456,789,101112), 10 epochs, batch 4, lr 5e-5, cosine, warmup 0.1, wd 1e-5, adamw_torch, bf16, cutoff 2048, val_size 0.1, load_best_model_at_end (no early-stop callback → all 10 epochs run). Realized full-FT is a one-line change: the `base/llama-3-8b-instruct/eval.yaml` config already uses `finetuning_type: full`, so full-FT train = base + `do_train: true` + DeepSpeed ZeRO-3. Wrote `configs/full_train.yaml.tmpl` and `configs/full_eval.yaml.tmpl` (disabled `push_to_hub`, which the paper sets to org rbelanec; eval points `model_name_or_path` at the trained checkpoint since full FT has no adapter). Mapped each dataset→metric (macro-F1 / binary-F1 / Pearson+Spearman / ReCoRD-EM / accuracy / CodeBLEU). Corrected cost: full fidelity ≈ $1.7–1.9k and ~350 instance-hrs (~2 wks on 1×2×H100, ~4 days on 4), dominated by 5 datasets (mnli/qqp/qnli/record/mmlu = 76% of examples). Wrote `reports/cost_estimate.md`. Safe levers: 3 seeds + early stopping → ~$700–1,150; data/epoch capping breaks fairness vs the paper's PEFT numerators unless those cells are re-run.
- 2026-06-03: Planning — wrote plan to full-FT LLaMA-3-8B-Instruct on the 27 PEFT-Bench datasets to serve as the denominator for a "% of full-FT performance recovered vs adapter size" plot. Key decisions: use PEFT-Factory (LLaMA-Factory fork, the paper's own tooling) with `finetuning_type: full` so data/templates/metrics match the paper by construction; provision on vast.ai with a **standard PyTorch image** (not the project's Nix `deterministic-serving` image — that's SM_90-only with a custom entrypoint, so the H100-only constraint and CUDA-symlink fixes in CLAUDE.md don't apply here); 2× H100 80GB + DeepSpeed ZeRO-3 as default (full FT of 8B needs ~130 GB, doesn't fit one 80 GB card). Flagged the main risk: paper's lr=5e-5 was tuned for PEFT and will likely overfit/diverge for full FT — plan includes an LR sweep {5e-6,1e-5,2e-5} in the smoke phase. Open questions for collaborators: recovery metric definition (raw ratio vs gain-normalized), dataset scope (all 27 vs plot subset), seed count (3 vs 5).
