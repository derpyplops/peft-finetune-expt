- 2026-06-03: Smoke-test debugging on the H200 (contract 39288289 on ssh8.vast.ai:18288 — first box 39286495 was destroyed for a vast "remote port forwarding failed" proxy bug; recreated on a different host). Four real issues found and fixed, each would have silently wrecked the sweep: (1) `/usr/bin/time` absent in the image → dropped the GNU-time wrapper from run_smoke. (2) **datasets==4.0.0 dropped script-based loading** → `super_glue` (cb, boolq) and `neulab/conala` (conala) fail; audit of all 27 via streaming = 24 OK, fix cb/boolq → `aps/super_glue` (parquet, verified identical cols/data), conala still needs a parquet mirror (candidates found: AhmedSSoliman/CoNaLa, HydraLM/conala_standardized, codeparrot/conala-mined-curated — TODO verify cols intent/snippet). (3) **CUDA OOM at the AdamW optimizer step** — full-FT 8B + `adamw_torch` overflows one 140GB H200 because `torch._foreach_sqrt` allocates a transient ~64GB copy of optimizer state (96GB steady + 64GB temp). Fixed with `optim: adamw_torch_fused` (same math, in-place step) → 126GB used, fits. (4) **checkpoint I/O bottleneck** — a full-FT checkpoint is ~32-100GB and takes ~138s to write; paper's `save_steps:0.05` = ~20 saves/run ≈ 46min I/O/run (fine for tiny adapters, ruinous for full FT). Coarsened to `save_steps/eval_steps: 0.2` (5/run). After fixes: pipeline trains clean, ~900 tok/s pure-compute on short-seq copa (optimizer-overhead-bound; long-seq datasets will be faster/token). Still TODO: validate eval→predict→compute_metrics→label-parsing on a completed run; fix conala source; measure real long-seq throughput to firm cost. Config deviations from paper (all forced by single-GPU full-FT, none affect the metric): fused AdamW, gradient_checkpointing, coarser save/eval steps.

- 2026-06-03: Premortem — wrote `reports/premortem.md`. Central insight: the dangerous failures here are silent-wrong-answer, not crashes. Top risks: (1.1) full FT at the paper's lr 5e-5 diverges/overfits → "% recovered" >100%; (1.2) generation eval doesn't string-match label vocab → denominator artificially ~0 on classification; (1.3) numerator (paper/W&B) vs our denominator computed in a drifted env / different metric choice → apples-to-oranges; (1.4) full FT < base on gsm8k/code → gain-normalized ratio undefined. Conclusion: the planned smoke test (cb/svamp/conala) is necessary but insufficient — it must additionally (a) inspect raw generations for label parsing, (b) include one large + one long-context (2048) run, (c) reproduce 1–2 of the paper's PEFT cells under our stack to validate the environment. Added go/no-go gates. This supersedes the smoke-test scope in plan.md Phase 2.
